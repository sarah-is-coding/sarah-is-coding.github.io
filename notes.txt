If you want to update the gh-pages branch with the latest changes from your main branch, the typical process is to:

- Checkout to main branch and ensure it's up to date.
- Run npm run build on the main branch. This will create or update the dist directory on the main branch.
- Without switching branches, copy the dist folder to a temporary location outside of your project directory. For example: cp -r dist/* /tmp/dist/.
- Now, checkout to the gh-pages branch with git checkout gh-pages.
- Delete the old dist directory: git rm -rf *.
- Copy the new build from the temporary location back to the project directory on the gh-pages branch: cp -r /tmp/dist/* ..
- Commit the changes with git add . and git commit -m "Update build".

In these commands, replace /tmp/dist/ with the path to the temporary directory where you copied the dist folder.
